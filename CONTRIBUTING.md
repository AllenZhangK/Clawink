# Contributing To Clawink

Thanks for your interest in Clawink.

Clawink is currently in a **public preview** stage. This repository is public, but the core product code is still being opened in stages from the internal repository.

That means collaboration is welcome, but the contribution model is intentionally limited for now.

## What Helps Most Right Now

At this stage, the most useful contributions are:

- Product feedback
- Target use cases and business scenarios
- Integration requirements
- UX pain points from real systems
- Documentation clarity suggestions
- Translation improvements for public-facing docs

Please use:

- **Issues** for concrete problems, requests, and gaps
- **Discussions** for broader scenarios, product direction, and open-ended feedback

For security-sensitive reports, use the private reporting path described in [SECURITY.md](SECURITY.md).

## What To Expect In The Preview Stage

During the preview stage:

- The internal repository remains the source of truth
- Public releases are published through one-way sync
- The public repository is not yet the full source-code development surface

Because of that, not every code change proposed in the public repository can be merged directly here.

## Source Code Contributions

Before the broader code release:

- Small documentation improvements are welcome
- Translation fixes are welcome
- Product and release feedback is strongly encouraged
- Source-code patches may be reviewed, but they can be re-applied manually in the internal repository first

In practice, that means:

- A public pull request might not be merged directly, even if the idea is accepted
- Accepted changes may first land in the internal repository and come back through the next public sync

This is expected while the public code boundary is still being defined.

## How To Write Useful Feedback

The most helpful reports usually include:

- The kind of system you want Clawink to support
- The user goal you expect AI to complete
- The operational friction or workflow gap you want reduced
- The level of control, approval, and observability you need

Concrete scenarios are more useful than abstract feature requests.

## Future Contribution Path

As Clawink opens more code publicly, this document will expand to cover:

- Development setup
- Pull request rules
- Review expectations
- Module ownership and extension boundaries

Until then, treat this repository primarily as a public preview and feedback surface.
